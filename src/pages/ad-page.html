In this page you should assemble the code needed to show a basic ad with some setup. It is expected that you will refer to reference materials. Also, unlike other pages, it is expected that when you get the ad to load it will cause warnings or errors in the console. As long as the source is the ad or ad assets you should ignore those issues.

The following values are being added into this page to allow you to make ad calls:

  • Network Code: 701
  • Parent ad unit code: "wpni.business"
  • adUnitCode: "personal-finance"
  • Ad Slot ID: "slug_inline_bb"

Using these variables you can set up your ad request by using the `googletag` publisher tag in order to display an ad in the slug_inline_bb div on this page. It is expected that you will search for additional documentation on how to do this process. In fact, your ability to find, understand and enact processes based on external documentation is one of the things we intend to test here.

You should add key value targeting to the ad by setting custom targeting parameters for the key "test-ad" with the value "active". You'll know you have added it successfully when you can look in the network requests and see a request that starts with `https://securepubads.g.doubleclick.net/gampad/ads?` and contains the key and value.

We also want to monitor the ad call in the console. Create a GAM event listener for the `slotRequested` event and log the slot object to the console. Make sure your script checks that the slot that's being logged is the specific slot you set up.

Once you have the ad displaying you need to change the design of the page to be two columns, a wide column for the content in the `main` HTML element on the left of the screen and a smaller column of 350px width for the `aside` HTML element to go on the right of the screen. (See image below, it should look similar to that though it does not need to be pixel perfect.) There is a line in `ads-page.css` under which you can add additional CSS to, including overwriting existing styles. Here we are checking your understanding of CSS and HTML layouts and how to manipulate them.

Once you have achieved this layout we want to fix the position of the ad in the right sidebar. We want to start the ad in one position and then trigger it to move when the user scrolls it to the top of the viewport (a vertical scroll down of 370px). You should need to use Javascript to accomplish this. You can see a video of what this should look like below. For this portion of the test we're checking your ability to use vanilla Javascript to accomplish the type of standard DOM manipulation tasks we'll frequently have to build or analyze.

If you are running this page without a local server you can see both the above image and video in the `public` folder of this project.
